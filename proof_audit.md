# 强哥德巴赫证明草稿审查表

这个文件用于审查“强哥德巴赫猜想已经被证明”的草稿。审查目标不是判断文字是否像证明，而是判断每个关键断言是否已经被严格支持。

## 1. 主定理

- 声称证明的定理：
- 量词是否覆盖所有偶数 `N > 2`：
- 是否允许附加假设：
  - 无条件：
  - 条件于 GRH：
  - 条件于 Hardy-Littlewood 猜想：
  - 条件于其他未证明命题：

如果不是无条件证明，就不是强哥德巴赫猜想的完整证明。

## 2. 表示目标

对任意偶数 `N > 2`，草稿必须证明：

```text
exists prime p such that 2 <= p <= N / 2 and N - p is prime
```

需要检查：

- 是否真的构造或证明存在这样的 `p`。
- 是否只证明了候选数很多，却没有证明 `N - p` 也是素数。
- 是否只证明了“几乎所有”偶数，而不是所有偶数。
- 是否把“三个素数之和”或“素数 + 半素数”误当成了“两个素数之和”。

## 4. 大数部分

如果证明使用解析数论估计，需要明确：

- 从哪个阈值 `N >= N0` 开始成立。
- `N0` 是具体数值还是只写了“充分大”。
- 每个误差项是否有效可计算。
- 主项是否严格大于误差项。
- 是否排除了所有可能的例外模类。
- 是否存在未处理的端点或奇偶性问题。

常见失败点：证明了某个平均量为正，却没有推出每个单独偶数的表示数为正。

## 5. 小数部分

如果证明依赖计算验证，需要记录：

- 验证范围：`4 <= N <= B`。
- 代码位置：
- 素性测试方法：
- 是否有独立复核：
- 是否保存反例搜索或见证输出：
- 解析部分起点 `N0` 是否满足 `N0 <= B + 2`：

如果 `B < N0 - 2`，证明存在未覆盖区间。

## 6. 交集论证

很多错误证明会定义两个集合：

```text
A_N = {p <= N / 2 : p is prime}
B_N = {p <= N / 2 : N - p is prime}
```

目标是证明 `A_N` 与 `B_N` 非空相交。需要检查：

- 是否给出了 `|A_N| + |B_N| > N / 2` 这类能强制交集非空的论证。
- 如果没有大小相加论证，是否给出了等价强度的结构性论证。
- 是否只估计了 `|A_N|`，却没有控制 `B_N`。
- 是否假设素数在每个相关模类中均匀分布，但没有有效误差界。

## 7. 结论检查

只有全部满足时，才可把证明视为候选完整证明：

- 主定理无条件覆盖所有偶数 `N > 2`。
- 所有引理已经证明或引用了已证明定理。
- 所有常数和阈值有效且可核验。
- 小数计算区间和大数解析区间无缝衔接。
- 没有使用 GRH、Hardy-Littlewood 猜想、随机模型或未证明的素数间隔断言。
- 最终一步直接推出存在两个素数 `p` 和 `N - p`。
